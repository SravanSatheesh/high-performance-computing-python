4. Why is the performance improvement lower than with matrix multiplication?
The performance is lower because recursion creates many Python function calls and uses dynamic types. Numba optimizes numeric loops much better 
than recursive functions. In contrast, matrix multiplication uses numeric arrays in loops, which Numba can fully compile to machine code.

5. What types of functions and datatypes are not suitable for Numba?
Functions that are recursive, use Python objects (strings, lists of mixed types, dictionaries), or highly dynamic Python features (generators, 
closures, eval) are not suitable for Numba.

6. What types of Python code benefit most from Numba?
Code that is numeric-heavy, uses loops over arrays, and operates on floats or integers benefits the most. Examples include matrix operations, 
numerical simulations, or any repeated calculations that Numba can compile to optimized machine code.
