Reflection:

1. Exploring different numbers of timesteps (e.g., 1000, 5000, 10000):

   By changing the total simulation steps, we can observe how computation time scales with problem size.

2. Time saved by Numba:

   - For 1000 steps: The speedup is minimal because the JIT compilation happens on the first run.
   - For 5000 steps: Speedup ≈ 2.9125 / 0.0031 ≈ 940× faster.
   - For 10000 steps: Speedup ≈ 5.8452 / 0.0067 ≈ 872× faster.

   This shows that for larger simulations, Numba drastically reduces computation time, achieving nearly 900× faster execution than standard Python.

3. Why this problem is suitable for Numba:

   - It involves heavy use of nested loops.
   - All calculations are numeric and straightforward.
   - Data is stored in NumPy arrays with fixed types.
   - There is no use of Python objects like lists or dictionaries.
   
   In general, Numba performs best when loops dominate runtime and data types are consistent and numeric.

4. What happens if lists are used instead of NumPy arrays:

   - Numba may switch to object mode, which severely reduces performance.
   - Execution speed approaches that of regular Python.
   
   NumPy arrays are preferable because they provide fixed datatypes, contiguous memory, and allow Numba to apply low-level optimizations efficiently.
